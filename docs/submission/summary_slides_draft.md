## スライド2: 課題の背景と内容、研究の目的
- 日本の肝臓移植: 待機15年、年間248件、登録者14,330人
- 問題: 量子最適化だけでは解決できない。データが集まらない。
- 2つの問題: (1) プライバシー → データ共有不能 (2) 組み合わせ爆発 → 古典で解けない
- 目的: 暗号基盤 + 量子最適化の同時解決

## スライド3: 解決案の内容 — 開発した技術（1）アーキテクチャ
- Niobi 3層構造図:
  - hyde (PQC) → データを出しても読めない
  - plat (FHE) → 暗号化したまま計算
  - argo (ZKP) → 適合を証明、中身は非開示
  - Quantum (QUBO) → 最大マッチング
- 8ステッププロトコル概要

## スライド4: 解決案の内容 — 開発した技術（2）QUBO定式化
- 二値変数 x_{d,r}
- 目的関数・制約条件
- 適合性スコアの5要素: ABO, MELD, GRWR, 虚血, 待機
- ペア交換チェーン

## スライド5: 解決案の内容 — 評価方法・検証フロー
- Greedy vs Quantum (Simulated Annealing)
- N=5〜200のスケール比較
- Brute Forceとの一致検証（N≤8）
- Rust/WASM実装 → ブラウザで実行可能

## スライド6: 検証結果と考察
- ベンチマーク表: N=150で+12マッチ = +12人の命
- N=200: 200! ≈ 10^375（古典不可能）
- 「1マッチの見落とし = 1人の死」

## スライド7: 成果を踏まえた今後のビジョン
- 4層展開パス: 国内 → 二国間 → 地域連合 → グローバル
- 「コピーは無害」「拒否の不可視性」「恣意性ゼロ」
- ブロックチェーンの次: 不透明なまま信頼を構築
- 全国民プール → 全世界プール

## スライド8: 成果のポイント（1）量子有用性・課題解決貢献度
- 量子有用性: Greedy超え+12マッチ、N=200で古典不可能
- 課題解決: プライバシー+量子の同時解決 = 社会実装への唯一のパス
- 新規性: 世界初の4技術統合（FHE+ZKP+PQC+量子アニーリング）

## スライド9: 成果のポイント（2）経済・社会インパクト
- 社会: 待機中死亡の減少、恣意性排除、データ主権
- 経済: 2030/2040/2050の定量試算
- 他領域展開: 献血、治験、Scope3、AML、希少疾患

## スライド10: 成果のポイント（3）他課題/他領域への展開
- Niobiはプロダクトではなくインフラ
- TCP/IPのように1回実装すれば全分野に展開
