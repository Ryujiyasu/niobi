## [0:52–2:15] 実機デモ① プライバシー基盤（暗号化したまま全工程）
**【画面】** `cargo run --release --bin demo` を実行し、出力をスクロール。各ステップでテロップを重ねる。
**【テロップ】**（左上に「実機デモ ― このマシンで実行」を常時表示）
- ① 40人が各自の鍵を生成（3.2ms）― 共有する秘密はゼロ
- ② 各自の鍵で暗号化（0.8ms）― 残るのは暗号文だけ、中身は誰も読めない
- ③ 暗号文のまま適合度を計算 ― 400ペア中295組が適合、一度も復号しない
- ④ ゼロ知識証明（295件 / 151ms）― 血液型もMELDも明かさず「適合」を証明
- ⑤ 拒否は不可視 ― 沈黙と「未通知」が暗号学的に区別不能。だから1億人が入れる
- ⑥ 合意した二人だけ病院へ ― 病院は、どう探したかを一切知らない
- 全工程200ms。データは最初から最後まで暗号化されたまま

## 制作メモ
- **音声なし前提**：説明は全てテロップが担う。BGMは任意（無くても可）。テロップは背景に対し可読なコントラスト・サイズで。
- **尺**：約4:30。詰めるなら創薬パート（3:15–4:00）を字幕＋静止図に圧縮し、demo＋exchange_benchの2本を主役に。**領域単位審査では横断性が効くので創薬パートは残すのを推奨。**
- **exchange_qubo_benchの所要時間**：フル（N=8〜30×8シード）は数分。録画用は①事前実行の出力をスクロール、または②短縮版（小N・少シードで~15秒）を用意。短縮版が必要なら作成します。
- **画質/形式**：フルHD・MP4・実操作画面録画。
- **収録前チェック**：各デモを1回実行し、テスト数・実測msがテロップと一致するか確認（環境差で前後しうる）。プロンプト匿名化（PS1）を忘れない。
